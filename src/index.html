Introducing ⇐ Recall, the search engine based on bookmarks

Remember before the first search engines? The web was a bit of a nightmare. Poorly organised. Hard to browse. Difficult to find things.

Search engines made finding new pages on the web a lot better. Some things haven't changed though. Bookmarks are still the way they were in the mid-nineties. You still organise them by hand. If you have a lot then you have a lot of organising.

Recall is a search engine for your bookmarks. You save bookmarks to it and then you can search through them. Unlike bookmarks in a web browser, you don't have to keep things meticulously organised. And the search is based on the body text of a link, not just the title or the url. If you have a lot of bookmarks, like I do, it's a real improvement. I don't bother filing bookmarks into sub-subfolders or making sure I give them all meaningful names. I just bung it all in and then search for what I want. Much easier.

It really is dead simple. You just drag and drop a button into your web browser and then you're ready to go. Use the button to bookmark pages on the web as you browse. It'll pop up a dialog box confirming things and then you click save. After you've added a page, Recall goes and downloads it, analyses it and saves the analysis to its "search index" ready for you to search for it later.

The important part is that the whole body text from the page is indexed. It doesn't just use the title of the page - using all the text makes searching a lot better.

When you search, Recall looks through the whole body text of every page you've ever bookmarked and shows you the relevant ones. That way, if you have a bunch of bookmarks on gardening, you can just search for "gardening" and it'll show them all. Personally, I have the Recall search page set as my home page in my browser - it's the page I use the most.

I think everyone has been annoyed by a bad site search engine. My local library's search engine drives me round the bend. The search is slow and then when things appear they're basically irrelevant. Recall is different. It's built on top of the Lucene text search library. This is the same text search library used by Facebook, Microsoft and Twitter. It's state of the art.

Recall is a subscription product - there is an annual fee. This makes it a little different from traditional "dot-com" web apps. They let anyone sign up for free. Then they make their money by selling customer data to advertisers. Sometimes the adverts are so crafty, it's hard to tell what's real and what's an advert. I don't want to be doing that, so I charge the annual fee. It means I don't have to sell my soul (or yours!) to marketers.

Probably, you already have some bookmarks saved. They can be imported. Recall supports something called the "Netscape Navigator" bookmark file format. It's as old as the hills and every web browser is able to export to it. If you have different bookmarks saved on different computers, you can import them all. It takes about a minute.
